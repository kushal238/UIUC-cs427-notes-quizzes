核心概念
Multiplicity（多重性）— 读法规则
Multiplicity 0..1 1..* 0..* 1..4
多重性数字写在连线的对面——数字写在 A 侧，表示的是一个 B 对应多少个 A。
读法口诀
看哪边的数字 → 就在约束那边的数量
即：数字在 A 侧 = "从 B 的角度看，B 拥有多少个 A"
符号含义例子
1恰好 1 个（默认值，可省略不写）每个 City 恰好在 1 个 State
0..1零个或一个（可选）员工可以有 0 或 1 个直属上司
0..*零个或多个（无限制）Author 可以没有任何 Book
1..*至少一个Order 至少包含 1 个商品
1..41 到 4 个每篇 Article 有 1~4 个 Author
实战解读
Quiz 图例解析
Article Author Book Publication
连线两端的多重性需要分别独立解读：
连线数字位置解读
Article ↔ AuthorAuthor 侧：1..4每篇 Article 需要 1~4 个 Author
Article ↔ AuthorArticle 侧：0..*一个 Author 可以写 0 到多篇 Article
Book ↔ AuthorAuthor 侧：0..*每本 Book 可以有 0 到多个 Author
Book ↔ AuthorBook 侧：0..*一个 Author 可以写 0 到多本 Book
关键推论
Author 侧是 0..* → "An author may have written nothing" ✅ 是正确的。
Book 侧是 0..* → "Not every Book has an Author" ✅ 也是正确的。
因此 "Every publication has an Author" ❌ 是错误陈述。
Class Diagram 基础
类图的结构与用途
Class Diagram Attribute Operation Static Structure
Class Diagram 是 UML 中描述系统静态结构的图——展示类、属性、方法及类之间的关系。
Class（类）
系统中的实体或概念，用矩形框表示
Attribute（属性）
类的数据特征，如 name、age
Operation（操作）
类的行为/方法，如 order()、calculate()
Class Diagram 的局限
类图只能表示静态结构，不能表示：
  • 算法逻辑（algorithmic nature）
  • 时序 / 时间信息（timing-related information）
  • 对象间的交互过程
建模实践
识别 Class vs. Attribute vs. Operation
Class Attribute Operation
从需求描述中提取建模元素的判断方法：
元素类型特征识别方式
Class持久化的实体/概念名词，可被实例化，有自己的属性
Attribute描述类的数据特征名词，从属于某个类，描述"它是什么"
Operation类的行为或动作动词，描述"它能做什么"
Pizza 餐厅例子
需求：顾客可以下单比萨或饮料，比萨有大小、酱料、底料
  • Class：Pizza、Customer、Drink、Sauce
  • Attribute：Size（大/小）、Base（底料种类）
  • Operation：order()——描述"下单"这个动作，而非持久化实体
Order 在此场景中 最不可能是 Class，更可能是 Operation。

Sequence Diagram Lifeline Message Time Triggering Order
Sequence Diagram 描述对象之间随时间推移的交互顺序，是 UML 行为图（Behavioral Diagram）的一种。
水平轴
参与交互的 Objects（对象），每个对象有一条生命线
垂直轴
Time（时间），从上到下时间递增
水平箭头
Message（消息），对象间的调用或回复
事件顺序依据
时序图中事件排列顺序由 Triggering Time（触发时间） 决定——谁先被触发，谁就排在上方。

考点总结
Week 5 高频考点速查
易错点
  • Part-whole 关系用 Aggregation 或 Composition 表示，不是 Generalization
  • Generalization = 继承（is-a），箭头从子类指向父类，箭头是空心三角
  • 抽象类类名在 UML 中用斜体表示，无法直接实例化
  • 多重性数字写在对面，约束的是它所在那侧的数量
  • Class Diagram 不能表示时序和算法；要用 Sequence Diagram
  • Sequence Diagram 垂直轴 = Time（时间），水平排列的是 Objects
  • 事件顺序由 Triggering Time 决定，不是重要性或持续时长
  • Order 在餐厅例子中是 Operation，不是 Class
  • Composition（实心菱形）：部分离开整体就消失；Queue 内嵌 Array 是典型例子
