Design Patterns
Creational 创建型模式
关注对象如何被创建
Singleton
唯一实例 · 全局访问点
Factory Method
子类决定实例化哪个类
Abstract Factory
创建一族相关对象的接口
Builder
分步构建复杂对象
Prototype
克隆现有对象创建新对象
Structural 结构型模式
关注类与对象如何组合
Composite 重点
树形结构 · 统一处理个体与群体
三角色
Component 接口 Leaf 叶节点 Composite 容器
⚠ 子节点管理方法（add/remove）属于 Composite 类，不在 Component 接口中
Adapter
接口适配 · 兼容不兼容的接口
Decorator
动态附加职责到对象
Facade
提供统一的高层接口
Behavioral 行为型模式
关注对象交互与职责分配
Observer 重点
状态变化 → 自动通知所有订阅者 Event notification across multiple listeners
Strategy 重点
封装算法族 · 运行时互换
Interpreter 重点
文法解析 · 构建 AST
Interpreter 角色
Terminal 叶节点 NonTerminal 内部节点 Context 传递状态
⚠ AST 由 Expression 实例构成，Context 不是树节点
Iterator
顺序遍历集合元素
Visitor
不改类结构，添加新操作
State
状态驱动行为变化
核心考点
考点 01
一个类可参与多个模式
A class can participate in more than one design pattern
考点 02
三大分类定义不能混淆
Creational 创建 · Structural 结构 · Behavioral 行为
考点 03
Strategy vs Visitor
Strategy 替换算法整体；Visitor 在不改类的前提下新增操作
场景 → 模式 速查
3D 对象库，单个与组合需要相同变换操作
→ Composite Pattern
对冲基金需 on-the-fly 切换交易策略
→ Strategy Pattern
状态变化时需要自动通知多个依赖方
→ Observer Pattern
需要解析自定义语言或表达式文法
→ Interpreter Pattern
顺序访问集合元素，不暴露内部结构
→ Iterator Pattern
两个不兼容接口需要协同工作
→ Adapter Pattern
